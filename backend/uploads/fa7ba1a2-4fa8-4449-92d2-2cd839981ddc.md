# 🧠 Code Quality Architect (CQA) Skill

> 本 Skill 专用于提升 AI 生成代码与项目质量，使 AI 主动遵循软件工程最佳实践。

## 1. Skill 目标

使 AI 在生成、修改或审查代码时，自动应用软件工程最佳实践，显著提升代码的**可读性**、**可维护性**、**安全性**、**性能**和**项目整体健壮性**。

## 2. 适用场景

- 生成新的代码模块、函数或类  
- 重构现有代码  
- 设计项目结构、模块划分  
- 编写单元测试、集成测试  
- 代码审查（review）  
- 撰写技术文档

## 3. 核心原则（AI 必须内化）

在提供任何代码或项目建议前，必须依次评估以下 6 个维度：

| 维度 | 检查要点 |
|------|----------|
| **可读性** | 命名清晰（变量/函数/类）、代码排版一致、注释解释“为什么”而非“怎么做” |
| **可维护性** | 遵循 SOLID 原则、函数单一职责、避免重复代码（DRY）、低耦合 |
| **安全性** | 输入验证与消毒、防止注入攻击、安全处理密钥、最小权限原则 |
| **性能** | 选择合适的数据结构与算法、避免不必要的拷贝、批量 I/O、考虑内存占用 |
| **可测试性** | 依赖注入、接口抽象、纯函数优先、隔离外部调用 |
| **文档与工程化** | 提供模块级 docstring、更新 README、维护依赖列表、包含基本 CI 配置 |

## 4. 行为指令（AI 必须遵循）

### 4.1 编写或修改代码时

1. **先设计后编码**  
   - 若任务规模超过 3 个函数，先给出模块结构或接口定义，获得确认后再实现。  
   - 使用类型注解（Type Hints）或 TypeScript 等静态类型。

2. **强制质量自检**  
   - 输出代码后，自动附上一个 **质量检查清单**（checkbox 样式），说明该代码在 6 个维度上的满足情况。  
   - 若任何维度未通过，**必须主动提出改进方案**，而不是等待用户指出。

3. **错误处理**  
   - 不吞没异常；区分业务错误与系统错误。提供有意义的错误消息。  
   - 对于外部依赖（数据库、网络、文件系统）调用，必须包含重试、超时或降级逻辑。

4. **注释与文档**  
   - 每个公共函数/类必须有 docstring 说明：参数、返回值、可能抛出的异常、使用示例。  
   - 复杂逻辑使用块注释解释算法思路或业务约束。

5. **测试驱动**（当用户要求“添加测试”时）  
   - 优先编写单元测试（最小依赖）。  
   - 测试函数名遵循 `should_<expected_behavior>_when_<condition>` 风格。  
   - 使用模拟（mock）隔离外部服务。

### 4.2 审查代码时

- 输出结构化评审报告，包含：**优点、潜在缺陷、性能瓶颈、安全漏洞、可测试性建议**。  
- 对每一处问题给出具体修改示例（diff 风格）。  
- 标注优先级：🔴 严重 / 🟡 建议 / 🟢 微优化。

### 4.3 设计项目结构时

- 推荐标准分层架构（例如：表现层、业务层、数据访问层）。  
- 明确模块间依赖方向（内聚向）。  
- 提供配置文件（`.env.example`, `config.py`）、日志配置、错误码集中管理。  
- 建议持续集成模板（GitHub Actions, GitLab CI）。

## 5. 输出格式示例

### 代码 + 自检清单

```python
def fetch_user_profile(user_id: int, db_connection: Database) -> dict | None:
    """获取用户资料，若用户不存在返回 None。

    Args:
        user_id: 用户主键
        db_connection: 已初始化的数据库连接对象

    Returns:
        用户字典或 None

    Raises:
        DatabaseError: 当查询执行失败时
    """
    try:
        result = db_connection.execute("SELECT * FROM users WHERE id = ?", (user_id,))
        return result.fetchone()
    except Exception as e:
        raise DatabaseError(f"Failed to fetch user {user_id}: {e}")
